# PRD：内置提词器功能（Teleprompter）—— 录音节奏同步版

> **版本**：v1.0  
> **状态**：已评审 / 待进入 UI + 技术方案评审  
> **编写日期**：2026-05-21  
> **目标产品**：专业屏幕录制工具 / 独立提词器模块（macOS First，v1 仅支持 macOS 13+）  
> **核心创新**：用户通过一次「练习录音」即可让提词器精确复刻自己真实的说话节奏，而非依赖手动滑块或 AI 猜测。

---

## 1. 产品概述

### 1.1 背景与定位
当前市面上的视频录制工具（FocuSee、Screen Studio、Loom、OBS 等）在提词器功能上普遍存在两个痛点：
- 纯手动速度滑块：无法准确匹配个人语速、停顿、节奏变化，尤其在长文稿场景下极易读丢、读快或读慢。
- 依赖 AI 自动跟读（部分竞品规划中）：存在隐私风险、延迟、准确率不稳定，且与「本地优先、专业创作者」定位冲突。

本产品定位为一款**「录制即专业输出」**的屏幕录制工具（或可作为独立模块集成）。我们将提词器从「辅助显示工具」升级为**「个人节奏复制器」**：
- 用户先输入文稿 → 进行一次练习录音（真实念稿）
- 系统通过「云端高精度 ASR + 本地轻量文本对齐」的混合引擎，生成精确的**个人节奏时间戳映射表**（用户可完全掌控是否使用云端）
- 后续正式录制时，提词器按照用户**自己**的说话节奏自动、平滑、连续滚动（重点支持「连续两行」自然呈现）

这使得提词器真正成为「第二条声带」，让创作者在正式录制时能以最自然、最自信的状态完成讲解。

### 1.2 目标版本范围
- **v1（MVP）**：完整闭环「文稿输入 → 练习录音 → 云端/本地混合节奏映射 → 正式录制按节奏滚动」，支持手动微调对齐
- **v1.5**：节奏可视化编辑器、多脚本管理、滚动过程中实时微调
- **v2**：多练习录音融合、断点自动检测、导出节奏模板供他人使用（可选）

**v1 核心策略**：练习阶段采用「云端高精度 ASR 对齐（优先）+ 本地兜底」的混合方案，正式录制阶段 100% 本地、无任何云端调用。用户在练习录音前会收到清晰的授权提示，仅练习音频与文稿会短暂上传用于生成节奏映射，映射结果与正式视频均严格本地存储。

## 2. 目标用户与核心场景

### 2.1 核心用户画像
| 用户类型 | 典型场景 | 核心痛点 | 使用频次 |
|----------|----------|----------|----------|

| 在线讲师 / 培训师 | 企业内训、在线课程、知识付费 | 需要自然语速、停顿、强调，不能像念经 | 高 |
| 产品经理 / 增长黑客 | 产品演示、客户提案、销售 Demo | 脚本迭代快，需要快速多轮练习后一镜到底 | 中高 |
| 营销 / PR | 品牌故事、广告脚本、直播带货 | 要求专业、自然、带感情 | 中 |
| 学生 / 研究者 | 答辩、学术报告、实验演示 | 首次上镜，极度紧张 | 中 |

### 2.2 Jobs-to-be-Done（用户要完成的工作）
1. **「我不想背稿，但又要讲得像自己写的」** —— 通过练习录音把「我的节奏」固化下来
2. **「我录到一半读丢了位置」** —— 阅读引导线 + 精确节奏滚动双保险
3. **「我语速时快时慢，固定速度滑块不管用」** —— 节奏映射自动适应所有变化
4. **「我不想让观众看到我在看稿」** —— 提词器内容绝不进入录制画面（最高优先级）
5. **「练习了三遍，正式录制还是不对味」** —— 一次练习即可复用，多次练习可融合优化

### 2.3 核心使用场景（优先级排序）
1. **长形式教程 / 在线课程**（最核心场景）
2. **产品功能演示 + 讲解**（脚本 + 操作同步）
3. **销售 / 客户提案视频**
4. **品牌故事 / 年度复盘视频**
5. **学术答辩 / 技术分享**

## 3. 价值主张与差异化

### 3.1 核心价值主张
> **「一次练习，终身复用自己的声音节奏」**

不是「让机器猜你想怎么说」，而是「把你已经说过的自然节奏，精确复制到下一次」。

### 3.2 三大差异化（必须全部实现）
| 差异化点 | 说明 | 竞品现状 |
|----------|------|----------|
| **录音节奏同步引擎** | 用户练习录音一次，即可生成个人专属时间戳映射表，后续滚动完全复刻个人语速、停顿、呼吸 | FocuSee 仅支持手动速度滑块；Screen Studio 无提词器 |
| **零可见脚本录制** | 提词器窗口采用多层窗口/捕获排除机制，100% 不进入最终画面 | 大部分工具要么没有，要么实现不彻底 |
| **极致本地化** | 所有音频处理、文本对齐、节奏存储 100% 本地，永不上传 | 部分竞品计划使用云端 Whisper |

### 3.3 成功愿景（用户语言）
- 「我第一次用就觉得像在跟自己对话」
- 「录完之后我都不敢相信这是我第一次正式录」
- 「再也不用因为忘词重录 5 遍了」

## 4. 功能需求（分模块详细描述 + 用户故事 + AC）

### 模块 4.1 脚本输入与管理（Script Input & Management）

**功能描述**  
用户可在提词器入口处输入或导入完整讲解文稿，支持纯文本和基础 Markdown（标题、列表、加粗）。系统自动保存草稿，支持多脚本管理、模板库、最近使用。

**用户故事**
- 作为 UP 主，我想直接粘贴之前写好的 Notion/Word 稿子，不想重新排版
- 作为培训师，我想把常用开场白做成模板，一键加载

**详细交互逻辑**
- 入口：录制器主界面「提词器」按钮 → 打开「脚本编辑器」模态/侧边面板
- 输入区：大文本域（支持 10,000+ 字），左侧实时字数统计 + 预计时长（基于上一次节奏）
- 支持操作：粘贴、导入 .txt/.md、导出、另存为模板、从历史脚本加载
- 自动保存：每 3 秒本地 IndexedDB / 文件系统
- 分段标记：用户可手动插入「---」或点击「添加断点」将文稿逻辑分段

**边缘情况 & 错误处理**
- 空脚本：禁用「开始练习」和「开始录制」按钮，提示「请先输入文稿」
- 超长文稿（>15,000 字）：给出性能提示，建议分段录制
- 导入文件编码错误：自动尝试 UTF-8 / GB18030 回退，失败则提示手动复制粘贴

**验收标准（AC）**
1. 用户粘贴 8000 字文稿后，3 秒内完成渲染，无卡顿
2. 点击「保存为模板」后，下次打开脚本编辑器可在「我的模板」中一键加载
3. 关闭编辑器后重新打开，文稿内容 100% 还原（含手动插入的断点）
4. 支持从剪贴板直接导入时，自动去除多余空行和 Markdown 注释

### 模块 4.2 练习录音与节奏映射引擎（Practice Recording & Timing Alignment）—— 核心模块

**功能描述**  
这是本产品的杀手级功能。用户在「练习模式」下真实念一遍文稿，系统同时录制音频（支持仅音频或 音频+屏幕），通过「云端高精度 ASR + 本地轻量文本对齐」的混合引擎生成精确的时间戳映射表（sentence/word → start/end timestamp）。后续正式录制时，提词器严格按照此映射表驱动滚动，实现「复刻自己声音」的效果。

**用户故事**
- 作为创作者，我想先小声练习三遍，把最舒服的节奏「录下来」，正式录制时完全按照这个感觉走
- 作为讲师，我想把「停顿 1.2 秒强调重点」这个个人习惯固化到提词器里

**详细交互逻辑**
1. **进入练习模式**：脚本编辑器中点击「开始练习录音」按钮
2. **练习录音界面**（全屏或浮动）：
   - 上方：当前正在念的句子高亮（可手动点击跳转）
   - 中间：大字号提词器（默认速度 1.0x，可实时调节但不影响最终映射）
   - 下方：波形可视化 + 实时音量条
   - 右侧：计时器、已念字数、当前段落进度
3. **录音控制**：Space 暂停/继续、R 重新开始本段、Esc 结束练习
4. **录音结束后自动对齐**：
   - 系统弹出「节奏映射生成中」（进度条 + 预计 5-15 秒，视文稿长度）
   - 使用本地引擎完成音频 → 文本时间戳对齐
   - 生成「节奏映射文件」（JSON，本地存储，与脚本绑定）
5. **对齐结果预览**：
   - 文稿每行/每句旁边显示时间戳（00:01.230 - 00:04.560）
   - 用户可播放任意句子验证对齐是否准确
   - 提供「手动微调」入口（见模块 4.3）

**对齐技术实现（v1 推荐方案）**
- 首选：macOS `SFSpeechRecognizer` + `AVAudioEngine` 时间戳 / Windows `Windows.Media.SpeechRecognition`（系统内置，零依赖）
- 次选：集成 `whisper.cpp` 本地二进制（离线，跨平台一致性更好）
- 兜底：波形可视化 + 用户手动拖拽对齐（纯交互，无算法）

**边缘情况 & 错误处理**
- 用户念漏/念错某句：对齐后该句时间戳为 null，用户可在预览界面手动标记或重录该段
- 背景噪音过大导致对齐失败率 >30%：提示「噪音较大，建议在安静环境重录」或进入手动对齐模式
- 练习录音时长与文稿字数严重不匹配（<0.3 字/秒 或 >5 字/秒）：给出警告「语速异常，请检查是否完整念完」

**验收标准（AC）**
1. 800 字文稿，安静环境下，练习录音后 8 秒内完成对齐，句子级时间戳准确率 ≥ 92%（人工抽检 20 句）
2. 对齐结果支持导出为标准 JSON（便于调试和未来导入）
3. 用户可针对单句「重新练习本句」而不影响其他句子的映射
4. 即使对齐失败，也能生成一个「均匀分布」的兜底映射表，保证功能可用

### 模块 4.3 节奏可视化编辑器（Timing Visual Editor）—— v1 必须

**功能描述**  
提供一个「波形 + 文本」双栏编辑器，让用户直观地查看和手动修正系统生成的节奏映射。这是「不依赖 AI」的关键保障。

**用户故事**
- 我发现第 3 段「因为……所以……」的停顿被系统识别成了两个句子，我想手动合并
- 系统把「嗯……」的思考停顿算进正文了，我想删除这个停顿的时间

**详细交互逻辑**
- 左侧：音频波形（可缩放、播放头）
- 右侧：文稿文本，每行/每句可折叠，旁边显示起止时间
- 交互：
  - 拖拽波形上的分割线 → 调整句子边界
  - 点击文本行 → 波形播放头跳转到对应位置
  - 按住 Shift 框选多行 → 批量「合并为一句」或「插入停顿」
  - 右键菜单：标记为「重音」「停顿」「过渡」
- 实时预览：修改后立即可点击「试播」用当前节奏滚动提词器（不录制）

**验收标准（AC）**
1. 拖拽分割线精度达到 10ms 级别
2. 支持撤销/重做 20 步
3. 「试播」功能 100ms 内响应，滚动平滑无卡顿
4. 保存后，正式录制时必须严格使用用户最终修正后的映射表

### 模块 4.4 滚动引擎（Dual-Mode Scrolling Engine）

**功能描述**  
同时支持两种滚动模式，用户可在正式录制前自由切换：
- **手动模式**：经典速度滑块 + 方向键微调（兼容 FocuSee 体验）
- **节奏同步模式**（默认推荐）：严格按照练习录音生成的时间戳映射表自动滚动

**详细交互逻辑**
- 速度滑块（0.5x - 2.0x）在节奏模式下变为「整体偏移」滑块（±30%），用于微调整体快慢而不改变相对节奏
- 每行/每句到达「阅读引导线」时自动高亮 + 轻微放大（用户可配置）
- 支持「连续两行」呈现策略：当前句 + 下一句同时显示在屏幕中央区域，当前句高亮，下一句半透明
- 滚动曲线：使用 ease-out cubic + 轻微惯性，模拟人类自然眼动

**边缘情况**
- 映射表中某句时长为 0：自动跳过或使用平均时长
- 用户在录制中途想临时加速：提供「临时 +20%」热键，松开后恢复原节奏

**验收标准（AC）**
1. 节奏模式下，正式录制时提词器滚动时间与练习录音时间偏差 ≤ 150ms（全程）
2. 手动模式下，速度滑块调节后 50ms 内滚动速度生效
3. 支持「跳转到下一段落」热键，跳转动画 ≤ 300ms

### 模块 4.5 视觉定制系统（Visual Customization）

**功能描述**  
完整复刻 FocuSee 的文本显示定制能力 + 额外支持「节奏模式」下的视觉提示。

**详细规格**
- 字体：系统字体 + 5 种专业阅读字体（可扩展）
- 大小：12px - 72px 连续可调，默认 28px（Mac 推荐）
- 颜色：文字颜色、背景颜色、引导线颜色（支持透明度）
- 透明度：0% - 100%（Mac 特别强调，Windows 通过 DWM 实现）
- 高对比模式：一键切换（WCAG AA 标准）
- 行高 / 字间距 / 段落间距
- 主题：浅色 / 深色 / 跟随系统 + 自定义主题

**验收标准（AC）**
1. Mac 上透明度 40% + 文字 80% 不透明度组合下，录制画面 0% 捕获到提词器内容
2. 长时间（>30 分钟）阅读后，用户主观疲劳度评分 ≤ 3/10
3. 所有设置实时生效，无需重启

### 模块 4.6 阅读辅助系统（Reading Guide & Focus）

**功能描述**  
除 FocuSee 提到的「阅读引导线」外，额外增加「节奏感知」辅助。

**核心组件**
- 可开关的 Reading Guide Line（当前行下方 1/3 位置的细线或箭头）
- 当前行「呼吸式」高亮（opacity 1.0 → 0.85 循环）
- 下一行预览区（可选，常开推荐）
- 段落进度条（顶部）
- 断点跳转按钮（上一段 / 下一段）

**验收标准（AC）**
1. 引导线位置精度：无论滚动速度如何，始终与当前句视觉中心偏差 ≤ 8px
2. 支持「仅高亮当前句」和「高亮当前+下一句」两种模式

### 模块 4.7 显示与窗口行为（Window & Display Behavior）

**功能描述**  
提词器必须在录制过程中保持「存在但不被录制」的状态。

**详细规格**
- 默认模式：浮动半透明窗口，可拖拽、可调整大小
- 始终置顶（但录制时可临时「穿透」给底层应用）
- 窗口位置/大小/透明度记忆（按脚本或全局）
- 录制中最小化控制条：仅显示「暂停滚动 / 速度 / 进度」三个控件
- 多显示器支持：提词器可放在副屏，主屏录制内容
- PiP / 侧边吸附 / 全屏三种布局

**关键技术要求**（见第 7 章）

**验收标准（AC）**
1. 提词器窗口被录屏软件捕获的概率 = 0%（在推荐捕获模式下）
2. 录制中拖拽窗口，底层被录画面无任何卡顿或闪烁

### 模块 4.8 快捷键与无障碍（Shortcuts & Accessibility）

**全局热键（录制中生效）**
- `Space`：暂停/继续滚动
- `↑ / ↓`：手动微调当前滚动位置（±1 行）
- `← / →`：整体速度 ±10%
- `[` / `]`：跳转上一/下一断点
- `Esc`：隐藏提词器（不停止录制）
- `Cmd/Ctrl + Shift + T`：全局唤起提词器

**无障碍**
- 完全键盘可操作
- VoiceOver / Narrator 友好（朗读当前句、进度）
- 高对比 + 大字体模式默认开启选项

**验收标准（AC）**
1. 所有热键在录制中 100% 响应，冲突率 ≤ 1%（与常见录制软件）
2. VoiceOver 下可完整完成「加载脚本 → 开始练习 → 正式录制」流程

### 模块 4.9 与录制流程的深度集成（Recording Workflow Integration）

**完整状态流转**
```
主界面「提词器」按钮
  ↓
脚本编辑器（输入/导入/练习）
  ↓
[练习录音] → 生成节奏映射 → 节奏编辑器（可选）
  ↓
返回主录制界面 → 选择「使用节奏模式」或「手动模式」
  ↓
开始正式录制 → 提词器自动进入「录制态」（最小化控制条 + 按节奏滚动）
  ↓
录制结束 → 自动收起提词器 → 弹出「是否保存本次节奏为新版本？」
```

**录制中提词器行为**
- 窗口自动切换为「录制专用外观」（边框变细、控件最小化、鼠标 3 秒无操作自动隐藏）
- 即使用户切换到其他应用，提词器窗口依然置顶且滚动正常
- 录制暂停时，滚动同步暂停

**验收标准（AC）**
1. 从点击「开始录制」到提词器开始按节奏滚动，延迟 ≤ 800ms
2. 录制过程中切换桌面/应用，提词器滚动不中断
3. 录制结束后 2 秒内提词器自动隐藏，不残留窗口

## 7. 技术实现建议与关键难点

### 7.1 平台优先级
- **v1 必须支持**：macOS 13 Ventura 及以上（Apple Silicon M 系列优先支持，Intel Mac 次之）
- **推荐技术栈**：Tauri 2.0（Rust + WebView）或 Electron + Rust 子进程（性能关键部分）
- 理由：需要同时操作本地音频、窗口管理、屏幕捕获排除，Web 技术 + 原生扩展最优

### 7.2 「提词器窗口不被录制」核心难点（最高优先级）

必须在 v1 解决，否则产品价值归零。

**跨平台实现方案对比**：

| 方案 | Mac 实现 | Windows 实现 | Pros | Cons | v1 推荐 |
|------|----------|--------------|------|------|---------|
| **ScreenCaptureKit Exclude** | `SCStreamConfiguration` + `excludedWindows` | N/A | 官方、零开销 | 仅 Mac | ★★★★★（Mac 首选） |
| **窗口样式 + 捕获过滤** | `NSWindow` + `setSharingType` | `WS_EX_LAYERED` + `SetWindowDisplayAffinity` | 跨平台 | 部分录屏软件可绕过 | ★★★★ |
| **虚拟摄像头 / 叠加层** | 复杂 | 需驱动 | 彻底 | 性能差、用户体验差 | 不推荐 |
| **Presenter-only / 辅助窗口** | `CGDisplayStream` 过滤 | `IDXGIOutputDuplication` 过滤 | 彻底 | 实现极难 | v2 研究 |
| **多显示器 + 副屏录制** | 用户手动 | 用户手动 | 简单 | 依赖硬件 | 兜底方案 |

**v1 推荐方案**：
- **Mac**：100% 使用 ScreenCaptureKit `excludedWindows`，提词器窗口在创建时即加入排除列表
- **Windows**：使用 `SetWindowDisplayAffinity(HWND, WDA_EXCLUDEFROMCAPTURE)`（Win10 2004+）
- 同时提供「手动切换到副屏」引导，作为终极兜底

**验证方法**：用 OBS + macOS 内置「屏幕录制」+ 第三方工具（Bandicam / ShareX）三重验证，捕获率必须为 0。

### 7.3 练习录音与节奏映射引擎（云端优先 + 本地兜底混合方案）

这是 v1 最核心的技术模块，直接决定「一次练习就能精准复刻个人节奏」的成败。

#### 7.3.1 整体策略（2026 推荐）

**优先级顺序**（用户可在设置中全局选择默认路径）：

1. **云端高精度路径（默认首选）**  
   把练习录音 + 用户完整原始文稿一起发送给支持「参考文本对齐」或「词级时间戳」的云端 ASR 服务，获得极高精度的句子/词级时间戳，再在本地做一次轻量「文本对齐修正」，把时间戳强映射回用户原始文稿。

2. **本地兜底路径**  
   当用户选择「完全离线」、网络失败、或云端返回质量过低时，自动/手动切换到 whisper.cpp（DTW 时间戳） + 本地对齐算法。

3. **纯手动兜底**  
   波形可视化编辑器（任何路径失败时均可进入）。

这种混合方案能在「体验极致」和「隐私可控」之间取得最佳平衡。

#### 7.3.2 云端路径详细实现（首选）

**推荐服务（2026 年中文场景排序）**：

| 排名 | 服务 | 词级时间戳精度 | 是否支持传参考文本 | 中文表现 | 价格参考（2026） | 备注 |
|------|------|----------------|--------------------|----------|------------------|------|
| 1 | OpenAI Whisper API (gpt-4o-audio / whisper-1) | 极高（支持 word + segment） | 支持 `prompt` 字段塞全文稿 | 优秀 | 按时长 | 最成熟，timestamp_granularities 参数 |
| 2 | 火山引擎（Volcengine）实时/非实时 ASR | 极高 | 支持「热词」+ 全文提示 | 顶级（中文） | 极低 | 国内延迟最低 |
| 3 | 讯飞开放平台（iFlyTek） | 高 | 支持「语义级」对齐 | 顶级（中文） | 低 | 适合带口音/方言 |
| 4 | AssemblyAI / Deepgram | 高 | 支持 `audio_url` + `transcript` | 良好 | 中 | 英文更强 |
| 5 | Azure Speech Services | 高 | 支持 `phraseList` | 良好 | 中 | 企业级 |

**v1 首推**：OpenAI Whisper API + 火山引擎作为国内备选。

**API 调用示例（OpenAI Whisper）**：

```http
POST https://api.openai.com/v1/audio/transcriptions
Authorization: Bearer $OPENAI_API_KEY
Content-Type: multipart/form-data

audio: [练习录音.wav]          # 16kHz 单声道，<25MB
model: whisper-1
language: zh
prompt: "以下是完整讲解文稿，请严格按照文稿用词和分句返回：\n[用户完整原始文稿全文]"
response_format: verbose_json
timestamp_granularities: ["word", "segment"]
temperature: 0.0
```

**关键 Prompt Engineering 技巧**（极度重要）：

- 把用户**完整原始文稿**作为 `prompt`，并明确指令「严格按照文稿用词和分句返回」。
- 额外加一句「不要添加、删除或改写任何内容」可进一步降低幻觉。
- 如果文稿很长（>3000 字），可先按「段落」切分多次调用，再合并时间戳（需处理跨段边界）。

**返回数据结构示例**（简化）：

```json
{
  "text": "...",
  "segments": [
    {"id": 0, "start": 0.82, "end": 4.15, "text": "大家好，今天我想和大家聊聊提词器..."},
    ...
  ],
  "words": [
    {"word": "大家好", "start": 0.82, "end": 1.23},
    {"word": "今天", "start": 1.45, "end": 1.78},
    ...
  ]
}
```

#### 7.3.3 本地后处理：文本对齐算法（把云端/本地输出映射到用户原始文稿）

无论云端还是 whisper.cpp 返回的结果，都**不能直接信任**，必须做一次「已知参考文本对齐」。

**推荐算法流程**（Rust / TypeScript 均可实现）：

1. **文稿预处理**：把用户原始文稿按「句」切分（标点 + 自定义 `---` 断点），得到句子数组 `original_sentences: string[]`
2. **云端/本地输出预处理**：把返回的 words/segments 展平成词序列 `recognized_words: {text, start, end}[]`
3. **序列对齐**：
   - 使用 **Levenshtein Distance（编辑距离）** 或 **Needleman-Wunsch 全局对齐** 计算最优匹配。
   - 对于每个 `original_sentences[i]`，找到它在 `recognized_words` 中对应的词片段。
   - 取该片段的最小 `start` 和最大 `end` 作为该句的最终时间戳。
4. **置信度打分**：
   - 编辑距离 / 长度比 < 0.15 → 自动接受
   - 0.15 ~ 0.35 → 标记「需确认」（在节奏编辑器中高亮）
   - > 0.35 → 标记「对齐失败」，强制进入手动编辑

**Rust 推荐库**：
- `whisper-aligner`（专门做 whisper 输出 → 参考文本对齐）
- `strsim` + 自定义 DP 实现 Levenshtein
- 可选：`rust-fst` 或 `tantivy` 做快速模糊匹配

**结果落盘格式**（与 PRD 之前定义一致）：

```json
{
  "scriptId": "uuid-v4",
  "practiceId": "uuid-v4",
  "source": "cloud-openai" | "local-whisper" | "manual",
  "createdAt": "2026-05-21T16:20:00Z",
  "segments": [
    {"text": "大家好，今天我想和大家聊聊...", "start": 0.82, "end": 4.15, "confidence": 0.97}
  ]
}
```

#### 7.3.4 错误处理 & 重试策略

- **云端返回质量差**（平均置信度 < 0.7 或编辑距离过高）：自动降级到本地 whisper.cpp 重跑一次，并提示用户「云端对齐精度较低，已切换本地引擎，可在编辑器中手动优化」。
- **网络超时/失败**：3 次重试（指数退避）→ 自动切本地。
- **用户中途取消上传**：立即停止上传，进入「纯手动对齐模式」，并记录「用户选择离线」。
- **文稿与录音严重不匹配**（用户只念了前半段）：检测到总时长与字数比异常时，弹出「是否仅对齐已念部分？」让用户确认。
- **敏感内容拦截**：上传前做简单本地关键词扫描（可选），发现「密码」「身份证」等提示「建议先删除敏感信息再对齐」。

#### 7.3.5 用户授权与隐私 UI 流程（必须实现）

在「开始练习录音」按钮旁边或弹窗中，必须出现**显式授权卡片**：

```
[ ] 我了解并同意：本次练习录音及文稿将发送至 [服务商名称] 进行节奏对齐，
    生成的节奏映射文件将 100% 本地保存，正式录制全程不涉及任何网络请求。
    [了解更多隐私说明]  [选择「完全离线模式」]
```

- 默认勾选「云端优先」，用户可一键切换「完全本地（whisper.cpp）」。
- 上传进度条必须显示「正在上传音频...」「正在对齐文稿...」「正在下载映射结果...」
- 对齐完成后，练习录音原始文件**默认自动删除**（用户可选择「保留练习录音用于复听」）。

#### 7.3.6 存储与清理策略

- 节奏映射文件：`~/Library/Application Support/[AppName]/rhythms/{scriptId}/{practiceId}.json`
- 练习录音（可选保留）：同目录下 `.wav`，默认 7 天后自动清理，或用户手动删除。
- 提供「一键清除所有练习数据」入口，彻底删除所有相关文件。

### 7.4 性能要求
- 滚动引擎：60fps+，CPU < 3%（M1 / i5 以上）
- 透明窗口开销：Mac Metal Layer < 8% GPU
- 长时间（2 小时）录制：内存增长 < 150MB
- 对齐速度：1000 字文稿 < 12 秒（M1 / i5）

### 7.5 隐私与本地化（更新版）

- **正式录制阶段**：100% 本地，提词器滚动、视频捕获、所有处理均不联网。
- **练习对齐阶段**（仅此阶段）：
  - 用户**显式授权**后，练习录音 + 完整文稿会短暂上传至所选云端 ASR 服务（OpenAI / 火山 / 讯飞等）。
  - 上传文件默认在对齐完成后**自动删除**（用户可选择保留）。
  - 节奏映射结果（.json）100% 本地加密存储。
- 所有文件位置：`~/Library/Application Support/[App]/rhythm/` 或 `%APPDATA%\[App]\rhythms\`
- 提供「一键清除所有练习数据」功能，彻底删除原始录音、映射文件及临时文件。
- 设置中常驻「默认对齐方式」开关：云端优先 / 完全本地（whisper.cpp） / 每次询问。
- 隐私政策必须单独说明「练习阶段云端调用」的范围、保留策略和服务商列表。

## 8. MVP 定义、里程碑与优先级

### 8.1 v1 MVP 必须包含（最小可上线闭环）
- 脚本输入（文本 + Markdown 基础支持）
- 练习录音（音频本地录制）
- **练习节奏映射引擎**：云端高精度 ASR（OpenAI/火山/讯飞等）+ 本地 whisper.cpp 兜底 + 轻量文本对齐算法
- 节奏可视化编辑器（波形 + 文本，可拖拽调整，支持置信度高亮）
- 两种滚动模式（手动 + 节奏同步）
- 完整视觉定制 + 阅读引导线
- 录制流程深度集成（最小控制条 + 零捕获）
- 全局快捷键 + 无障碍基础
- macOS 13+ 完整可用（Windows 支持计划在 v2）
- 清晰的云端授权 UI + 隐私开关（练习阶段可完全离线）

### 8.2 v1 不包含（明确排除）
- **实时/自主 AI 提词**（录制过程中 AI 自动听用户说话并驱动滚动、改写提示词等）
- 多脚本融合 / AI 节奏自动优化（v1.5+）
- 云同步、团队协作、云端存储节奏模板
- 富文本（图片、表格、颜色标注）
- 移动端 / Web 端

**注意**：v1 明确**允许**在「练习录音对齐阶段」调用云端 ASR（需用户显式授权），但**禁止**在正式录制阶段进行任何网络请求或实时 AI 行为。

### 8.3 v1.5 规划（3-4 个月后）
- 节奏编辑器增强（多练习录音加权平均）
- 滚动中实时「呼吸」微调
- 导出/导入节奏模板
- 脚本分段自动检测

### 8.4 v2 规划（半年后，可选）
- 实时语音驱动滚动（本地小模型）
- 多人协作节奏模板
- 与视频剪辑软件（DaVinci / Premiere）插件联动

## 10. 成功指标、风险与开放问题

### 10.1 成功指标（v1 上线 3 个月后）
| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 提词器功能激活率 | ≥ 35% | 埋点 |
| 练习录音后正式录制使用率 | ≥ 70% | 埋点 |
| 用户主观「一次成片率提升」评分 | ≥ 4.2 / 5 | 问卷 |
| 重录次数下降 | ≥ 40% | 问卷 |
| NPS（使用过提词器用户） | ≥ 55 | 问卷 |
| 零捕获验证通过率 | 100% | 内部测试 |

### 10.2 主要风险
1. **对齐准确率不足** → 导致用户反复手动调整，体验崩坏
   - 缓解：强兜底手动编辑器 + 多引擎 fallback
2. **Windows 捕获排除实现不彻底** → 部分录屏软件仍能抓到
   - 缓解：提供「副屏录制」强引导 + 文档详细说明
3. **用户不愿意做练习录音** → 觉得多一道工序
   - 缓解：强调「练习 1 遍 = 正式少重录 5 遍」，并提供「快速均匀节奏」一键兜底

### 10.3 开放问题（需团队决策）
1. 是否在 v1 同时支持「纯手动模式」（无练习录音也能用）？
2. 节奏映射文件是否加密？默认是否开启？
3. 是否允许用户导出节奏模板给他人（商业模式相关）？
4. 练习录音时是否同时录制屏幕？（可用于后期对齐校验，但增加存储）
