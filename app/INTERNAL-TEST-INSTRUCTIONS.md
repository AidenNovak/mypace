## 推荐的测试者画像（优先级排序）

1. **最优**：正在做中长视频口播 / 知识分享 / 课程录制，每周至少录 1-2 条视频的创作者。
2. **很好**：B 站 3-10w 粉，经常吐槽「忘词」「对稿麻烦」「看着镜头不自然」的 UP 主。
3. **可接受**：小红书 / 视频号 / YouTube 中文区中长视频创作者，有提词器痛点。
4. **避免**：纯剪辑党、几乎不自己出镜的人。

**目标数量**：8-15 人（质量 > 数量）

我在做一个 macOS 提词器工具，核心特点是：
- 录屏时完全看不到窗口
- 你先自然念一遍，它就能记住你的节奏，后面严格按你的语速停顿滚动 + 逐字高亮

目前做到 0.8 版本，核心节奏引擎和逐字跟读已经能跑了，想找 10 位左右真实在做口播/知识视频的创作者做小范围内部测试。

测试大概需要 15-25 分钟（装一下 → 体验隐形窗口 → 练习一次节奏映射 → 再念一遍感受）。

会送一个内部测试版的 DMG + 详细说明文档。测试后希望能收到你 5-10 分钟的真实反馈（文字或语音都行）。

如果你最近在准备视频稿，特别欢迎试用。

需要我发给你吗？随时可以退出的～

## 反馈收集方式建议

**推荐组合**：
- 主要用微信语音 / 文字（最自然）
- 或者小红书 / B 站私信
- 准备一个简单的飞书 / 腾讯文档问卷作为补充（可选）

**强烈建议让测试者直接发语音**（比填问卷更真实）。

## 执行 checklist（你可以直接按这个推进）

- [ ] 申请/确认 Apple Developer ID 证书（必须）
- [ ] 在本地配置好 `build-app.sh` 所需的几个环境变量
- [ ] 运行 `./build-app.sh notarize` 产出带公证的 DMG（或先用 ad-hoc 版给最熟的 2-3 人）
- [ ] 更新 `VLOGGER-README.md` 里的联系方式（底部）
- [ ] 准备 8-15 个目标 UP 主名单（可以先列 20 个）
- [ ] 开始一对一私信（建议每天 3-5 个，保持跟进）
- [ ] 每发出一份 DMG，就记录在表格里（姓名 + 平台 + 什么时候发的 + 是否已反馈）
- [ ] 收到反馈后及时回复感谢，并追问 1-2 个具体问题
