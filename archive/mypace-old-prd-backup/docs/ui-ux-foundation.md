# UI/UX 基础设计（Text Version）

> 本文档描述 MyPace 项目的视觉语言、核心界面布局、交互原则与设计约束，供贡献者参考。目标是打造「极简、专业、不打扰、长时间阅读不累」的专业提词体验。

## 设计原则（Design Principles）

1. **一次练习，终身复用**  
   所有交互都服务于「把用户自己的节奏精确复制下来」这个核心。

2. **录制中极致克制**  
   正式录制时，UI 必须最小化干扰。3 秒无操作 → 控件自动隐藏，只剩文字。

3. **零可见脚本**  
   提词器窗口内容绝不进入录制画面（最高优先级技术约束）。

4. **云端仅在练习阶段**  
   练习对齐时清晰授权，正式录制 100% 本地。

5. **可控 + 可回退**  
   任何自动对齐都必须能被用户轻松手动修正。

6. **长时间阅读不疲劳**  
   默认字号、行高、对比度都经过专业阅读场景优化。

## 视觉语言（Visual Language）

- **主色**：#3B82F6（专业蓝）—— 仅用于引导线、强调、按钮
- **背景**：浅色 #F8F9FA / 深色 #1F2937（跟随系统）
- **文字**：系统 UI 字体 + Inter / SF Pro，阅读字体可选 5 种
- **默认字号**：28px（Mac 推荐），12~72px 连续可调
- **透明度**：Mac 重点支持 0~100% 实时调节
- **圆角**：录制中控制条使用 12px 柔和圆角
- **动画**：滚动使用 ease-out cubic + 轻微惯性；高亮使用「呼吸式」opacity 循环（1.0 → 0.85）

## 核心界面概览（4 大界面）

### 1. 脚本编辑器（进入前）
- 左侧 60%：大号无边框文本编辑区（类似 Bear/Notion）
- 右侧 40%：可折叠设置面板（字体、颜色、透明度、滚动模式、快捷键提示）
- 底部：保存模板 + 开始练习 + 直接手动录制 三个主要动作
- 状态栏：字数 + 预计时长 + 当前版本

### 2. 练习录音界面
- 上方：当前句子高亮
- 中部：大字号提词器 + 实时波形 + 音量条
- 右侧：计时器、进度、已念字数
- 控制：Space 暂停、R 重录本段、Esc 结束

### 3. 浮动提词器（录制中）
- 无边框 / 极细边框
- 当前行：100% 不透明 + 轻微放大 + 呼吸高亮
- 下一行：65% 不透明（「连续两行」策略）
- 阅读引导线：#3B82F6 2px 细线
- 录制中：3 秒无操作自动隐藏所有控件

### 4. 最小控制条（录制中）
- 半透明圆角条（类似 macOS 控制中心）
- 仅显示：进度 + 当前句简短预览 + 暂停/速度微调/跳转按钮
- 可拖拽

### 5. 节奏可视化编辑器（练习后）
- 双栏布局：左侧波形（可缩放）+ 右侧文本卡片
- 支持拖拽分割线、Shift 多选、右键菜单标记
- 实时「试播」功能

## 交互模式

- **全局热键**（录制中生效）：Space / ↑↓ / ←→ / [ ] / Esc / Cmd/Ctrl+Shift+T
- **阅读辅助**：可开关的 Reading Guide Line + 下一句预览 + 段落进度
- **对齐质量可视化**：置信度高亮（绿色/黄色/红色），引导用户快速修正

## 无障碍 & 响应

- 完整键盘可操作
- VoiceOver / Narrator 友好
- WCAG 2.1 AA 高对比模式一键开启
- 多显示器支持（提词器可放副屏）

## 录制中最小干扰原则（最高优先级）

- 录制开始后自动切换「录制专用外观」
- 鼠标 3 秒无操作 → 所有控件消失
- 窗口始终置顶但不被捕获
- 任何操作都有即时视觉反馈，但绝不抢夺注意力
